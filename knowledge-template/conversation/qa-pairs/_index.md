# 💬 QA Pairs

> 每次 Agent 回复结束时自动生成 · 按日期 + 主题分类 · 完整全文（零截断）

## 目录结构

```
qa-pairs/
└── {YYYY-MM-DD}/               ← 日期目录
    ├── 🧠-概念解释/             ← 自动分类：概念/对比/定义
    ├── 🐛-问题排查/             ← 自动分类：bug/报错/修复
    ├── 💻-代码实现/             ← 自动分类：开发/实现/创建
    ├── 🔧-工具配置/             ← 自动分类：安装/配置/插件
    ├── 📊-架构设计/             ← 自动分类：设计/重构/方案
    ├── 🚀-性能优化/             ← 自动分类：性能/优化/加速
    └── 📖-其他/                 ← 兜底分类
```

## 文件格式

每个 `.md` 文件包含完整 Q&A 内容：

```markdown
# {提问前 200 字符}
> ⏱ 提问: {本地时间} | 回答: {本地时间} | 耗时: Xm Ys
> 🔗 会话: {session_id}

{Agent 完整回答 · 零截断}
```

## 写入机制

| 路径 | 触发 | 说明 |
|------|------|------|
| **Stop hook** | 每轮 Agent 回复结束 | 主路径，实时写入 |
| **SessionEnd flush** | 会话退出 | 备份路径，跳过已存在文件 |

## 跳过捕获

- 提问中包含 `[nc]` 标记 → 不写入本条
- `暂停捕获` → 暂停整个会话的写入
- `恢复捕获` → 恢复正常写入

## 主题分类规则

基于提问内容的关键词匹配，优先级从上到下，首个命中即为分类。无匹配则归入 `📖-其他`。
