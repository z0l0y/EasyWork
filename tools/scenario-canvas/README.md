# 🎨 EasyWork Scenario Canvas — 可视化技能编排画布

> 基于 React Flow（SVG 渲染，30k+ GitHub Stars）的拖拽式技能编排工具。
> 把你想要执行的 EasyWork 技能拖到画布上，用连线串联成完整工作流。

---

## 🧠 这是什么？

EasyWork 有 28 个技能（项目理解、代码审查、测试、复盘…），平时你只能**一次用一个**，或者走**固定的流水线**。

画布让你**自由组合技能**——就像搭乐高：

```
📐 项目理解 ──→ 🔬 代码追踪 ──→ 📋 总结报告
                  ↘
                    🔍 代码审查 ──→ 📦 提交拆分
```

每个方块是一个**技能节点**，箭头是**数据流**（上游的产出变成下游的输入）。

### 三栏说明

| 面板 | 位置 | 作用 |
|------|------|------|
| **技能面板** | 左侧 240px | 列出全部 28 个技能，按阶段分组。拖拽或双击添加到画布 |

| **属性面板** | 右侧 280px | 点击节点后编辑其配置：技能类型、标签、阶段、JSON 配置、交互暂停提示词 |

## 📦 节点是什么？

每个节点代表 EasyWork 一个**技能**。节点上的信息：

```
┌──────────────────────┐
│ 📐 项目理解           │  ← 头部 = emoji + 技能名，颜色按阶段区分
├──────────────────────┤
│ ← architecture       │  ← 输入端口（左）：这个技能需要什么数据
│ ← suggested_trace... │
│                      │
│ architecture →       │  ← 输出端口（右）：这个技能产出什么数据
│ suggested_trace... → │
│                      │
│ 💬 HITL              │  ← 交互标记（可选）：执行到这里暂停等待用户输入
└──────────────────────┘
```

### 节点颜色含义

| 颜色 | 阶段 | 含义 | 典型技能 |
|------|------|------|---------|
| 🔵 蓝色 | **理解 (Understand)** | 阅读、分析、理解输入 | 项目理解、需求理解、论文阅读 |
| 🟢 绿色 | **深挖 (Deep-dive)** | 深入追踪、拆解复杂问题 | 代码追踪、需求拆解 |
| 🔴 红色 | **实现 (Fix)** | 写代码、改文件、生成内容 | 代码实现、文档编写 |
| 🟡 橙色 | **验证 (Verify)** | 审查、测试、选型、审计 | 代码审查、测试验证、技术选型 |
| 🟣 紫色 | **总结 (Summarize)** | 画图、报告、复盘、拷打 | 架构可视化、总结报告、CTO拷打 |
| 🟠 橘色 | **交付 (Deliver)** | 提交、确认、知识沉淀 | 提交拆分、变更确认、知识沉淀 |

### 端口 = 数据契约

每个技能定义了自己的**输入端口**（需要什么）和**输出端口**（产出什么）：

- **输入端口**（左侧圆点）：这个技能执行前需要的上游数据。比如「代码审查」需要 `code_changes`（代码变更）
- **输出端口**（右侧圆点）：这个技能执行后产出的数据。比如「代码审查」产出 `review_report`（审查报告）

**连线时，把上游的输出端口拖到下游的输入端口**，数据类型匹配才能形成有效工作流。

## 🚀 快速上手

### 1. 启动

```bash
cd tools/scenario-canvas
npm install   # 首次需要安装依赖
npm run dev   # 自动打开浏览器 http://localhost:5173
```

### 2. 搭建你的第一个工作流

```
① 从左侧「🧠 理解类」拖 "📐 项目理解" 到画布
② 从「🔬 深挖类」拖 "🔬 代码追踪" 到画布
③ 从「🟣 总结类」拖 "📋 总结报告" 到画布
④ 从「项目理解」右侧 architecture 端口 → 拖线到「代码追踪」左侧 entry_function 端口
⑤ 从「代码追踪」右侧 trace_report 端口 → 拖线到「总结报告」左侧 findings 端口
⑥ Ctrl+S 保存 → 得到 YAML 场景文件
⑦ 在 EasyWork 中执行：/easywork:scenario run my-scenario
```

### 3. 典型场景模板

| 你想做什么 | 拖哪些技能 | 怎么连 |
|-----------|-----------|--------|
| 🔍 **理解新项目** | 项目理解 → 代码追踪 → 架构可视化 | architecture → entry_function → architecture |
| ✏️ **修复 Bug** | 需求理解 → 代码实现 → 代码审查 → 提交拆分 | requirement → requirement → code_changes → code_changes |
| 🧪 **质量检查** | 代码实现 → 测试验证 → 清单审计 → 总结报告 | code_changes → code_changes → checklist_report → findings |
| 📊 **技术选型** | 技术雷达 → 技术选型 → 总结报告 | radar_report → options → findings |
| 🧠 **复盘** | 总结报告 → 复盘分析 → CTO拷打 | summary_report → incident → report_to_review |

## ⌨️ 操作速查

### 鼠标操作

| 操作 | 效果 |
|------|------|
| 从面板**拖拽**到画布 | 添加技能节点（放到鼠标位置） |
| **双击**面板中的技能 | 添加节点（随机位置） |
| 在画布上**拖拽节点** | 移动节点位置 |
| 从**右端口拖线**到另一节点左端口 | 建立连线（数据流） |
| 点击节点 | 选中，右侧显示属性面板 |
| 点击画布空白处 | 取消选中 |
| **滚轮** | 缩放画布 |
| **中键拖拽** / Ctrl+左键拖拽 | 平移画布 |
| Shift+点击节点 | 多选节点 |

### 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+S` | 保存场景（下载 YAML 文件 + 复制到剪贴板） |
| `Ctrl+O` | 打开文件选择器加载场景 |
| `Ctrl+N` | 新建空白画布 |
| `Delete` / `Backspace` | 删除选中的节点和连线 |

## 💾 保存 & 加载

### 保存格式

保存为 EasyWork 标准场景 YAML：

```yaml
scenario:
  id: my-scenario
  name: my-scenario
  version: "1.0"
  canvas:
    strategy: quality-first
    nodes:
      - id: n1
        skill: read-project
        label: 理解项目
        phase: understand
        depends_on: []
      - id: n2
        skill: trace-code
        label: 追踪调用链
        phase: deep-dive
        depends_on: [n1]
    edges:
      - from: n1
        to: n2
```

保存的文件可以通过 `/easywork:scenario run <id>` 在 EasyWork 中执行。

### 加载

- 点击工具栏「📂 加载」按钮
- 选择 `.yaml` 或 `.json` 文件
- 画布自动还原节点位置和连线

## ✅ DAG 校验

点击「▶️ 校验」检查以下问题：

| 检测项 | 说明 |
|--------|------|
| 🔴 循环依赖 | A→B→C→A 形成死循环，无法执行 |
| ⚠️ 断开节点 | 有节点没有任何连线（可能是遗漏） |
| ⚠️ 缺失输入 | 技能有必填输入端口但没有上游节点提供数据 |
| ⚠️ 空画布 | 没有任何节点 |

## 🛠 技术栈

| 组件 | 用途 |
|------|------|
| [React Flow](https://github.com/xyflow/xyflow) (~30k⭐) | SVG 节点编辑器核心 |
| React 19 | UI 框架 |
| Vite 6 | 开发 & 构建 |
| 原生 CSS | 暗色主题，零 CSS 依赖 |

### 为什么选 React Flow 而不是 LiteGraph.js？

| | LiteGraph.js (旧版) | React Flow (新版) |
|---|---|---|
| 渲染 | Canvas2D → 模糊、DPR 问题 | **SVG** → 矢量清晰 |
| 拖拽 | 手动拦截 DOM 事件 | **HTML5 Drag API** 原生支持 |
| 连线 | 基础直线 | **Smoothstep 曲线 + 动画 + 箭头** |
| 社区 | ~8k stars | **~30k stars** |
| 维护 | ComfyUI fork，需手动 hack | **全职团队维护** |
