# 🧭 给 EasyWork 使用者的建议

> **AI 时代最宝贵的不是代码，是你脑子里的思维链。**
>
> AI 都会写代码了，还要程序员干嘛？要的就是这个——知道为什么干、干什么、怎么干、干完会怎样。

---

## 为什么要有这个文件夹？

EasyWork 有 28 个技能、29 条命令、点线网三级编排。但**技能再多，也替代不了你脑子里那根清晰的思维链**。

这个文件夹不讲"怎么用 EasyWork"，讲的是**使用 EasyWork 之前你应该先建立的思维习惯**。它不是操作手册，是思维手册。

## ⚡ 30 秒核心要义

```
做任何事之前，先回答五个问题：

1. 为什么要干这个？（原因）  ← 不回答这个，方向可能就错了
2. 具体要干什么？（动作）  ← 不回答这个，边界就会模糊
3. 打算怎么干？（方法）    ← 不回答这个，执行就会失控
4. 想要什么结果？（结果）  ← 不回答这个，就没法验收
5. 会有什么影响？（后果）  ← 不回答这个，就是在埋雷

然后，把这五个问题放在时间轴上：
• 事前：想清楚再动手
• 事中：边做边验证
• 事后：复盘 → 沉淀 → 改进
```

## 🔗 与 EasyWork 技能的关系

| 思维环节 | 可以配合的技能 |
|----------|---------------|
| 理解原因（Why） | `/easywork:requirements` `/easywork:project` `/easywork:trace` |
| 明确动作（What） | `/easywork:decompose` `/easywork:compare` |
| 选择方法（How） | `/easywork:implement` `/easywork:diagram` |
| 验证结果（Result） | `/easywork:examine` `/easywork:review` `/easywork:coverage` |
| 评估影响（Impact） | `/easywork:retro` `/easywork:selfcheck` `/easywork:meta` |

> 💡 技能是**执行工具**，思维链是**驾驶工具的人**。这个文件夹教你当好驾驶员。
