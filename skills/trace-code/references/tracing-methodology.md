# 代码追踪方法论

> 用于 trace-code 技能 · 深度追踪调用链

## 追踪策略

### 正向追踪（从输入到输出）
```
用户输入 → 入口函数 → 中间处理层 → 数据存储/返回
```
适用场景：理解完整业务流程

### 反向追踪（从输出到输入）
```
最终结果 ← 数据来源 ← 上游函数 ← 触发条件
```
适用场景：Bug 定位、理解"这个值从哪来"

### 扩散追踪（横向搜索）
```
核心函数 → 所有调用者 → 被调用者 → 同类模式
```
适用场景：重构影响分析、安全审计

## 追踪记录模板

```markdown
## 调用链: [起始函数]

1. [文件:行号] `functionA()` → 接收 [参数]，做 [处理]
2. [文件:行号] `functionB()` ← 被 A 调用，传入 [数据]
3. [文件:行号] `functionC()` ← 被 B 调用，查询 [表/服务]
4. [文件:行号] 返回 [结果] → 回到 B → 回到 A → 输出给用户

## 关键发现
- 数据转换: [输入类型] → [中间类型] → [输出类型]
- 副作用: [文件写入/网络调用/状态变更]
- 风险点: [无校验/并发不安全/硬编码]
```

## 常见陷阱

- ❌ 只跟踪正常路径 → 也需覆盖异常/超时路径
- ❌ 假设函数名=功能 → 必须读实现确认
- ❌ 忽略中间件/拦截器/装饰器 → 这些可能改变数据流
