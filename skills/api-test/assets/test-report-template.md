## 3. 非预期结果排查指南

### 3.1 按实际状态码分类

#### 如果返回 500（预期非 500）
| 可能原因 | 排查步骤 | 解决方案 |
|---------|---------|---------|
| 空指针异常 | 查看服务日志，搜索 `NullPointerException` | 修复空值判断 |
| 数据库连接超时 | `mysql -h {host} -P {port} -u {user} -p` 测试连通性 | 检查 DB 连接池配置 |
| 第三方服务超时 | 查看调用链追踪（Jaeger/Zipkin） | 增加超时时间或加熔断 |

#### 如果返回 400（预期 200）
| 可能原因 | 排查步骤 | 解决方案 |

| 参数校验生效范围不对 | 检查 Controller 的 `@Valid`/`@Validated` | 确保校验注解在正确的位置 |
| 类型不匹配 | 检查请求体 JSON 的字段类型 | 修正传入的参数类型 |

## 4. 错误码映射表

| 状态码 | 错误码 | 含义 | 触发条件 | 排查步骤 | 解决方案 |
|--------|--------|------|---------|---------|---------|
| 400 | INVALID_PARAM | 参数校验失败 | 约束违反 | 查看响应 `details` 字段定位具体参数 | 修正参数 |
| 401 | UNAUTHORIZED | 未认证 | Token 缺失/无效 | `curl -H "Authorization: Bearer $TOKEN" {health_check_url}` | 刷新 Token |
| 403 | FORBIDDEN | 无权限 | 角色不匹配 | 查询用户角色 + 接口所需权限 | 授权或换用户 |
| 404 | NOT_FOUND | 资源不存在 | ID 错误或已删除 | `SELECT * FROM {table} WHERE id={id};` | 用正确 ID |
| 409 | CONFLICT | 资源冲突 | 唯一键/状态冲突 | 查询资源当前状态 | 先查后操作 |
| 422 | UNPROCESSABLE_ENTITY | 语义错误 | 业务规则违反 | 检查字段间业务关联 | 按业务规则修正 |
| 500 | INTERNAL_ERROR | 服务器错误 | 代码异常 | 查看服务日志 | 修复代码 |
