# Historical Version Coverage（历史版本覆盖）

> **铁律 #36**：版本索引条目必须在受影响节点中有对应的独立版本小节。不可仅有版本索引条目 + 节点内一行摘要。产出覆盖矩阵表——版本/受影响节点/已覆盖节点/缺失节点/恢复状态。有缺失→阻断写入，先修复覆盖。
> 此闸门解决 v2.11 之前的一个隐蔽问题：版本索引看起来完整（v1,v2,v3都有），但节点内只有一句话摘要，内容实际上丢失了。

---

## 1. 问题定义

### 什么是"版本索引完整但节点内容不完整"

```
版本索引：
| v3 | 2026-06-21 | 修复 Safari 兼容性 | READ, CODE, REVIEW, EXAMINE |
| v2 | 2026-06-20 | 降低 bcrypt cost   | CODE, EXAMINE              |
| v1 | 2026-06-18 | 初始实现            | 全部                       |

节点内容：
## 2. CODE
### v3 — 2026-06-21（变更：Safari CSS fallback）
{完整内容 ✓}

### v1 — 2026-06-18（初始）
{完整内容 ✓}
                                    ← v2 缺失！但版本索引说它影响了 CODE
```

版本索引声明 v2 影响了 CODE 节点，但 CODE 节点下找不到 `### v2` 小节。这就是"版本覆盖缺失"。

### 典型缺失模式

| 模式 | 表现 | 常见原因 |
|------|------|---------|
| **索引有、节点无** | 版本索引表有 v{N} 行，但对应影响节点内无 `### v{N}` 小节 | Overwrite 时丢失旧版本小节 |
| **只有索引一行摘要** | 节点内有 `### v{N}` 但内容仅一行"修复了 X 问题" | Quick Fix 追加了占位但没有填充内容 |
| **内容在孤儿更新记录块** | v{N} 内容存在于 `## 📝 更新记录` 块（v2.8 遗留）| v2.8→v2.10 迁移不完整 |
| **跨节点不一致** | v3 在 READ 有完整内容，在 CODE 只有一行 | 只迁移了部分节点 |

## 2. 覆盖矩阵定义

| 列 | 含义 | 来源 |
|----|------|------|
| **版本** | 版本号（v1, v2, v3...）| 版本索引表 |
| **受影响节点** | 版本索引声明的"影响节点"列 | 版本索引表 |
| **已覆盖节点** | 实际在节点内找到 `### v{N}` 小节的节点 | 扫描文档 |
| **缺失节点** | 受影响节点 - 已覆盖节点 | 算法计算 |
| **恢复状态** | 缺失节点内容的可恢复性 | 算法判断 |

### 恢复状态说明

| 状态 | 含义 | 自动修复？ |
|------|------|----------|
| **complete** | 所有受影响节点都有对应版本小节 | —（无需修复） |
| **recoverable** | 缺失节点内容可从其他地方找到（孤儿更新记录块/round_report/step_outputs） | ✅ 可自动迁移 |
| **partial** | 缺失节点内容只有摘要级信息（如只有一行描述），可恢复但不完整 | ⚠️ 标注"摘要恢复"（触发铁律#37） |
| **lost** | 缺失节点内容完全找不到 | ❌ 标注"丢失"，记录在覆盖矩阵中 |
| **not_affected** | 版本索引声明影响了该节点但实际上不需要 | ⚠️ 修正版本索引 |

## 3. 检测算法

### Algorithm: CHECK_VERSION_COVERAGE

```
Input: 已解析的文档拓扑（来自 Topology Gate）
Output: version_coverage_matrix

1. 从版本索引表提取所有版本号 → versions[]
2. 对每个 version in versions:
     a. 从版本索引获取该版本的"影响节点"列表 → affected_nodes[]
     b. 初始化 covered_nodes = []
     c. 对每个 affected_node in affected_nodes:
         在文档中搜索节点 {affected_node} 下的 `### {version}` 子节
         if 找到 且 内容 > 100字（非空占位符）:
             covered_nodes.append(affected_node)
         elif 找到 但 内容 < 100字:
             标记为 "partial"——只有摘要
         else:
             标记为 "missing"
     d. 计算 missing_nodes = affected_nodes - covered_nodes
     e. 对每个 missing_node:
         搜索内容来源（孤儿更新记录块/round_report/step_outputs）
         if 找到可恢复的内容:
             标记为 "recoverable" / "partial"
         else:
             标记为 "lost"
3. 输出覆盖矩阵表
```

### Algorithm: REPAIR_VERSION_COVERAGE

```
对每个可恢复的缺失节点:
  - recoverable: 从孤儿更新记录块/step_outputs提取内容→迁移到对应节点下→新增 `### v{N}` 小节
  - partial: 同上，但追加声明 "⚠️ 该版本内容为摘要恢复，非逐字原始记录"（触发铁律#37）
  - lost: 在缺失节点下新增 "### v{N} — 内容已丢失" 占位，标注恢复状态为 lost
  
对索引不一致的情况:
  - 如果版本索引声明的"影响节点"与实际情况不符 → 修正版本索引
  - 如果节点内有额外版本小节但索引未列出 → 更新版本索引
```

## 4. 覆盖矩阵输出格式

```markdown
## 📊 历史版本覆盖矩阵（Historical Version Coverage）

| 版本 | 受影响节点 | 已覆盖节点 | 缺失节点 | 恢复状态 |
|------|-----------|-----------|---------|---------|
| v3 | READ, CODE, REVIEW, EXAMINE | READ, CODE, REVIEW, EXAMINE | — | ✅ complete |
| v2 | CODE, EXAMINE | EXAMINE | CODE | ⚠️ recoverable → 已从孤儿更新记录块迁移 |
| v1 | 全部 | READ, CODE, REVIEW | EXAMINE | ❌ lost（原始 EXAMINE 产出丢失） |

### 缺失节点详情

| 版本 | 缺失节点 | 恢复来源 | 恢复后内容长度 | 恢复后状态 |
|------|---------|---------|-------------|----------|
| v2 | CODE | `## 📝 更新记录` 孤儿块 → 迁移到 `## 2. CODE ### v2` | 850 字 | ✅ recoverable |
| v1 | EXAMINE | 无可恢复来源 | — | ❌ lost |

### 恢复后覆盖统计

| 状态 | 修复前 | 修复后 |
|------|--------|--------|
| complete | 2/4 版本 | 3/4 版本 |
| recoverable | 1/4 | 0/4（已迁移） |
| lost | 1/4 | 1/4（v1 EXAMINE） |
```

## 5. Gate Judgment Rules

| 深度 | 闸门类型 | 命中处理 |
|------|---------|---------|
| **detailed** | **HARD GATE** | 有缺失节点且 recoverable → 自动修复后重新检测（最多2轮）；有 missing 且 lost → 阻断写入，标注"内容丢失" |
| **standard** | **SOFT GATE** | 警告但允许继续（标注"版本覆盖不完整"） |
| **brief** | **SKIP** | 跳过但标注"历史版本覆盖未经检查" |

---

## 6. 反模式

- ❌ **版本索引列了影响但节点内无对应小节**：v3 声明影响了 CODE+EXAMINE，但 EXAMINE 下只有 v1
- ❌ **节点内只有一行摘要替代独立版本小节**：`### v3 — 修复了登录超时`→没有说明怎么修的、为什么这样修
- ❌ **孤儿更新记录块不迁移**：v2.8 遗留的 `## 📝 更新记录` 块仍然存在，内容未迁移到节点
- ❌ **覆盖矩阵显示全部 complete 但实际有缺失**：矩阵造假——手动检查节点内容可发现
- ❌ **旧版本小节被新版本覆盖**：v2 的内容被 v3 overwrite 成了 v3 的内容——v2 的历史就丢了
