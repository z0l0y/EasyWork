# 检查项编写模板

> 用于 checklist 技能 · 规范化检查项定义

## 检查项标准格式

```markdown
### [检查项编号] [检查项名称]

- **检查内容**: [一句话描述检查什么]
- **检查方法**: [具体操作步骤 / 工具 / 命令]
- **通过标准**: [明确的可量化标准]
- **失败处理**: [不通过时做什么]
- **风险等级**: L0 / L1 / L2 / L3 / L4
- **关联铁律**: [对应的铁律编号]
```

## 示例

```markdown
### C-01 代码风格一致性

- **检查内容**: 新增代码的命名/缩进/注释风格是否与项目现有代码一致
- **检查方法**: 运行 `pre-commit run --all-files`，人工对比相邻文件的风格
- **通过标准**: linter 零报错 + 人工抽查 3 处以上风格一致
- **失败处理**: 运行 auto-formatter 修复，重新检查
- **风险等级**: L1
- **关联铁律**: 铁律 #3（保持现有风格）
```

## 检查项分类

| 类别 | 适用场景 | 示例 |
|------|---------|------|
| 代码质量 (C-) | CODE 步骤后 | 风格/命名/复杂度 |
| 安全性 (S-) | REVIEW 步骤中 | 注入/XSS/权限 |
| 测试 (T-) | EXAMINE 步骤后 | 覆盖率/回归/边界 |
| 文档 (D-) | SUM 步骤前 | CHANGELOG/API文档 |
| 部署 (P-) | ASK 步骤前 | 回滚/迁移/灰度 |
