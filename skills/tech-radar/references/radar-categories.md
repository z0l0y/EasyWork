# 技术雷达分类体系

> 用于 tech-radar 技能 · 四象限 + 四环级定义

## 四象限

| 象限 | 含义 | 示例 |
|------|------|------|
| **技术** (Techniques) | 开发方法/实践/模式 | TDD, DDD, Event Sourcing |
| **工具** (Tools) | IDE/CLI/库/平台 | VS Code, Docker, Kubernetes |
| **语言与框架** (Languages & Frameworks) | 编程语言/Web框架 | Rust, Next.js, GraphQL |
| **平台** (Platforms) | 云服务/基础设施 | AWS Lambda, Vercel, Supabase |

## 四环级

| 环级 | 含义 | 决策 |
|------|------|------|
| **采纳 (Adopt)** | 成熟可靠，应广泛使用 | 新项目默认选择 |
| **试验 (Trial)** | 有潜力，值得在有限范围尝试 | 非关键项目可用 |
| **评估 (Assess)** | 值得关注，等待社区验证 | 仅研究，不用于生产 |
| **暂缓 (Hold)** | 已知问题或有更好替代 | 避免新投资 |

## 评估维度

每个技术条目从以下 4 个维度评分（1-5 分）：

1. **成熟度**: 社区规模/文档质量/生产案例数
2. **适用性**: 与团队技术栈的契合度
3. **学习成本**: 团队上手的时间投入
4. **维护风险**: 长期维护的人力/工具链风险

## 雷达更新策略

- **每季度**: 全量复查所有条目
- **每月**: 新增条目 + 环级变更
- **每周**: 社区动态监控（GitHub trending / HN / 技术博客）
