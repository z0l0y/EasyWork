# 测试覆盖率分析框架

> 用于 test-coverage 技能 · 覆盖率评估与改进

## 覆盖率维度

| 维度 | 含义 | 目标 |
|------|------|------|
| **行覆盖** (Line) | 哪些行被执行 | ≥ 80% |
| **分支覆盖** (Branch) | if/else 两边都测到 | ≥ 70% |
| **函数覆盖** (Function) | 每个函数至少调用一次 | ≥ 90% |
| **路径覆盖** (Path) | 关键业务流程全覆盖 | 核心路径 100% |

## 覆盖率盲区清单

- [ ] 错误处理分支（try/catch 的异常路径）
- [ ] 边界条件（空数组、null 输入、最大值+1）
- [ ] 并发/竞态条件
- [ ] 超时/重试逻辑
- [ ] 配置变更路径（feature flag 切换）

## 改进策略

1. **先补核心路径** → 再补异常路径 → 最后补边界条件
2. **优先补分支覆盖低的模块**（高风险区域）
3. **不追求 100% 行覆盖**（有些防御性代码无测试价值）
