# EasyWork 交互式新手引导

> **Version**: v2.7 | Agent 在首次检测到用户使用 EasyWork 时（或用户说"带我入门"时），
> 按以下脚本引导。此为对话脚本，非静默文档——Agent 应分步询问、等待回答后再继续。

---

## 阶段 0：检测首次使用

触发条件（任一满足即视为首次使用）：
- `.claude/easywork/workflow.log.jsonl` 不存在或为空
- 用户说"第一次用"、"新手"、"入门"、"教程"
- Agent 在上下文中未找到任何本次 session 的 EasyWork 执行记录

---

## 阶段 1：欢迎与了解需求

```
👋 欢迎使用 EasyWork！

EasyWork 是一套 AI 辅助开发工作流——它会帮你做任务分类、步骤裁剪、
代码审查、提交拆分等等，让你从"随缘写代码"变成"按流程走"。

但在开始之前，我想先了解你的情况：

1. 你现在想做什么？（修 Bug / 加功能 / 看代码 / 重构 / 写文档 / 其他）
2. 这个项目是一个人开发还是有团队？
3. 项目有自动化测试吗？（Jest / Pytest / Go test 等）
```

**Agent 等待用户回答后继续。**

---

## 阶段 2：推荐成熟度级别

根据用户回答，推荐合适的成熟度级别：

| 用户情况 | 推荐级别 | 话术 |
|---------|---------|------|
| 个人项目 / 原型 / 学习 | L1（4 个技能） | "你的项目现阶段适合 L1——包含需求理解、代码实现、代码审查和人工确认。不会太复杂，但能覆盖最重要的环节。" |
| 有 CI/CD 的正式团队 | L2（7 个技能） | "你的团队有测试和 CI，适合 L2——在 L1 基础上加入测试执行、提交拆分和总结。这是性价比最高的级别。" |
| 20+ 人 / 合规要求 | L3（10 个技能） | "你的团队规模和复杂度适合 L3 完整版——加入架构图、深度复盘等，适合需要完整审计链的场景。" |

**Agent 等待用户确认级别后继续。**

---

## 阶段 3：演示一个最小任务

用当前项目的一个极简任务（1 行改动），演示完整流程：

```
好，让我用一个小小的演示让你感受一下 EasyWork 是怎么工作的。

我们以"把配置文件里的超时时间从 5 秒改成 10 秒"为例——

🔍 任务分类：微调（⚡ 低风险，4 步）
📋 裁剪方案：
  [ ] 1. READ  — 理解你要改什么 ✅
  [ ] 2. CODE  — 改那个数字 ✅
  [ ] 3. REVIEW — 确认只改了该改的，没顺手碰别的 ✅
  [ ] 4. GIT   — 生成提交消息 ✅
  [skip] 5-9   — 全跳过（改一行配置不需要画架构图）

看到了吗？EasyWork 知道这不是一个需要深度审查和测试的任务。
但如果是修 Bug，它会自动加上测试、总结、复盘。

→ 现在你想正式开始你的第一个任务吗？告诉我你想做什么。
```

---

## 阶段 4：首任务实战

Agent 按正常 EasyWork 流程执行用户的第一个任务，但在以下节点额外解释：

1. **任务分类后**：解释为什么选了这个类型、跳过了哪些步骤
2. **干跑预览后**：解释"这是执行预览，像点外卖前确认订单——现在改方案不花钱，执行后再改就费劲了"
3. **REVIEW 完成后**：解释"刚才我做了七维度自审查。自己查自己是最容易放水的环节，所以还有二次独立抽查"
4. **工作流结束后**：输出学习总结

---

## 阶段 5：学习总结

首任务完成后输出：

```
🎉 你的第一个 EasyWork 任务完成了！

回顾一下这个流程：
- 任务类型：{类型}（{N} 步）
- 执行了：{实际执行的步骤列表}
- 跳过了：{跳过的步骤及原因}
- 发现的问题：{审查/测试发现的问题数}

📚 想了解更多？
- `skills/fullchain-dev-workflow/assets/walkthrough-example.md` — 3 个完整示例
- `QUICKREF.md` — 30 秒速查卡片
- `skills/fullchain-dev-workflow/references/gotchas.md` — 项目特定踩坑记录（Agent 会自动追加）

💡 小技巧：
- 说"直接用 L1 模式"可以临时降级
- 说"跳过测试"可以手动裁剪步骤
- 说"保存进度"可以生成断点快照

→ 下次需要 EasyWork 时，说"用 EasyWork {你的任务}"就行。
```

## 特殊场景

### 用户已经熟悉其他 AI 工作流

如果用户说"我懂这些概念"、"我之前用过 Copilot/AgentForge/Ring"：

跳过阶段 1-2，直接进入阶段 3（最小演示）。话术简化为：

```
你既然有经验，我就不啰嗦了。EasyWork 的核心区别是：
1. 任务分类前置——改文案不会走 9 步
2. 强制自检——每步产出必须对照 data-contract
3. 干跑预览——执行前能先看方案

用一个 1 分钟演示感受下？还是直接开始你的任务？
```

### 用户不耐烦

如果用户说"直接开始吧"、"跳过教程"：

```
好的，教程随时可以回来看——说"带我入门"就行。
现在告诉我你的任务吧。
```

## 跟踪首次完成

Agent 在第一个任务完成后向 `.claude/easywork/` 写入一个标记文件：

```json
{"onboarding_completed": true, "completed_at": "ISO 8601", "level": "L2"}
```

后续 session 检测到此标记后不再触发新手引导。
